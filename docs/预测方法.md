# 预测方法

本项目不使用“限制比分”的方式修正输出，也不手动禁止某些比分。

新的原则是：**输出概率最高的三个比分，这三个比分必须来自多来源加权共识**。

## 1. 先收集可靠预测来源

每场比赛预测前，优先收集多种可靠来源：

| 来源类型 | 作用 |
|---|---|
| 量化预测机构 | 提供模型化胜平负、比分或期望进球 |
| 赔率 / 市场共识 | 提供市场隐含概率和大小球方向 |
| xG / Elo / 数据模型 | 提供球队强弱、进攻和防守质量 |
| 专业媒体预测 | 提供明确比分或赛前判断 |
| 用户历史 CSV / Excel | 提供本地历史预测、真实赛果和校准数据 |

优先使用新、透明、和该场比赛强相关的来源。不要使用低质量 SEO 预测页、过期页面、没有清楚数据依据的预测。

## 2. 加权方式

默认权重：

| 类型 | 默认权重 |
|---|---:|
| 量化预测机构 | 30% |
| 赔率 / 市场共识 | 25% |
| xG / Elo / 内部统计模型 | 20% |
| 专业媒体或专家预测 | 15% |
| 用户上传历史数据 | 10% |

如果某类来源缺失，就把它的权重按比例分给其他可靠来源。

## 3. 如何得到三个比分

每个来源可能给出不同信息：

- 有的直接给比分，例如 2-1
- 有的给胜平负概率
- 有的给大小球概率
- 有的给 xG 或预期进球
- 有的只给球队强弱判断

Skill 需要把这些信息统一成候选比分评分：

1. 统计外部来源支持的比分。
2. 计算加权后的预期进球。
3. 计算加权后的胜平负方向。
4. 计算加权后的大小球和双方进球倾向。
5. 把候选比分与这些加权结果做匹配。
6. 按综合支持度排序，输出前三个比分。

这三个比分**不一定是同一种胜负结果**。如果证据接近，可以同时包含主队胜、平局、客队胜。例如：

```text
1. 墨西哥 2-1 南非：14.2%
2. 墨西哥 1-1 南非：11.8%
3. 墨西哥 1-2 南非：8.6%
```

## 4. 不使用硬限制

不要写这种规则：

```text
如果大 2.5 高于 52%，就不要输出 1-0。
如果双方进球高于 55%，就不要输出 0-1。
```

这种方式会人为限制模型。

正确做法是：

```text
如果多个可靠来源和市场数据共同支持 1-0，就输出 1-0。
如果加权共识更支持 2-1，就输出 2-1。
如果来源分歧很大，也输出前三个比分，并降低置信度。
```

## 5. Poisson 的角色

Poisson 仍然可以用，但只能作为内部统计模型的一部分，不能单独决定最终比分。

它主要用于：

- 生成候选比分
- 估算大小球
- 估算双方进球
- 辅助判断比分是否和预期进球相符

最终比分列表必须由多来源加权共识决定。

## 6. 输出原则

默认输出三个比分，按综合概率从高到低排列。

只在用户明确说“只给一个比分”时，才输出一个主预测比分。
