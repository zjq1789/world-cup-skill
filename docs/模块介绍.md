# 模块介绍

本项目围绕“世界杯比分预测”设计，重点是让输出保持简洁，但背后的分析流程尽量完整。

## 1. 输入识别模块

负责识别用户给出的比赛信息。

支持输入：

- 文字：例如“预测世界杯：阿根廷 vs 法国”
- 图片：世界杯赛程截图
- 表格：CSV / Excel 历史数据或批量赛程

识别目标：

- 比赛双方
- 比赛时间
- 赛事阶段
- 常规时间 / 淘汰赛 / 决赛等上下文

## 2. 比赛状态模块

预测前必须先判断比赛状态。

状态分为：

- 未开赛：可以做赛前预测
- 进行中：不做赛前预测，可说明适合做实时走势分析
- 已结束：输出真实比分，不再预测

这一模块可以避免“赛后预测”的问题。

## 3. 联网数据模块

每次预测前必须联网查最新数据。优先查询：

- 官方赛程和比赛状态
- 最新阵容
- 伤病停赛
- 近期战绩
- 世界排名或 Elo
- xG / xGA
- 赔率或市场概率
- 天气、场地、旅行距离等上下文

如果某类数据查不到，需要在输出中标注为“数据缺口”。

## 4. 历史数据模块

用于读取用户上传的 CSV / Excel 数据。

常见数据包括：

- 历史真实赛果
- 历史预测记录
- 球队状态参数
- 球员状态信息

历史数据不替代联网数据，而是作为本地校准信息。

## 5. 预测模型模块

建议使用集成思路，而不是单一模型。

核心组合：

- Elo：判断整体强弱和爆冷风险
- Poisson：生成比分概率分布
- xG 修正：修正进攻和防守质量
- 阵容修正：考虑伤病、停赛、首发和关键球员状态
- 市场校准：用赔率或市场概率作为参考信号

## 6. 输出报告模块

输出要简洁，默认只给一个主预测比分。

推荐结构：

```text
主预测比分：阿根廷 2-1 法国

参考概率：
- 胜平负：阿根廷胜 39% / 平 28% / 法国胜 33%
- 大小球：大2.5 46% / 小2.5 54%
- 双方进球：52%
- 晋级概率：阿根廷 53% / 法国 47%
- 爆冷概率：18%
```

## 7. 结果回写模块

比赛结束后，将真实赛果写入 `match_results.csv`，并更新历史预测命中情况。

用途：

- 记录真实比分
- 比较预测比分和实际比分
- 更新球队进攻、防守、状态参数
- 为下一场预测提供校准依据
